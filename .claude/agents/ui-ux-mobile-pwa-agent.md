# UI/UX, Mobile & PWA Agent — Aniston HRMS

## Purpose
Audit Aniston HRMS frontend for responsive design issues, mobile APK WebView compatibility, PWA manifest correctness, accessibility basics (ARIA, keyboard nav), overflow problems, touch target sizing, z-index stacking, and bottom navigation visibility.

---

## Mobile Overflow Checklist
Test at 375px (iPhone SE), 390px (iPhone 14), 360px (Android standard):

- [ ] No horizontal scrollbar appears at any mobile viewport
- [ ] Tables: wrapped in `overflow-x: auto` container with `min-w-full` table
- [ ] Long employee names: truncated with `truncate` class + `title` attribute for tooltip
- [ ] Action buttons in table rows: collapsed to kebab menu (3-dot) on mobile
- [ ] Sidebar: collapses to hamburger icon at `md:` breakpoint (< 768px)
- [ ] Modal width: `w-full mx-4` or `max-w-sm` on mobile — not fixed `w-[600px]`
- [ ] Form fields: single column on mobile (`grid-cols-1 md:grid-cols-2`)
- [ ] Card grids: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4` responsive cascade
- [ ] GPS map view: minimum 280px height, responsive width
- [ ] Toast notifications: not overflowing on mobile, positioned `bottom-4`

---

## Touch Target Size Requirements
All interactive elements must have minimum 44×44px touch targets (Apple HIG + WCAG 2.5.5):

- [ ] Buttons: `min-h-[44px] min-w-[44px]` or `py-3` equivalent
- [ ] Checkboxes: wrapper div has `p-2` padding to expand touch area
- [ ] Table row action icons: wrapped in `<button className="p-2">` (32px icon + 8px padding = 48px)
- [ ] Sidebar nav items: `py-3 px-4` minimum
- [ ] Bottom navigation tabs: `h-16` minimum (64px)
- [ ] FAB (AI Assistant): `h-14 w-14` minimum (56px — Material Design standard)
- [ ] Close button on modals: `p-2` with 24px icon = 40px minimum (acceptable)

## Accessibility Audit (WCAG 2.1 AA Basics)

### ARIA Labels
- [ ] All icon-only buttons have `aria-label`: `<button aria-label="Delete employee">` 
- [ ] Form inputs have associated labels (not placeholder-only)
- [ ] Dialog/Modal has `role="dialog"`, `aria-modal="true"`, `aria-labelledby`
- [ ] Alert/Toast has `role="alert"` or `aria-live="polite"`
- [ ] Tab panels: `role="tabpanel"`, `aria-labelledby`

### Keyboard Navigation
- [ ] All interactive elements reachable by Tab key
- [ ] Focus visible: `:focus-visible` styles present (not `outline: none` globally)
- [ ] Modal: focus trapped inside modal when open (focus guard)
- [ ] Modal: Escape key closes modal
- [ ] Dropdown: Arrow keys navigate options
- [ ] Skip link: "Skip to main content" at top of page for screen readers

### Color Contrast
- [ ] Body text on background: min 4.5:1 contrast ratio (WCAG AA)
- [ ] Large text (>18px): min 3:1 contrast ratio
- [ ] Brand indigo-600 (#4f46e5) on white: check ratio (should be ~5.9:1 ✓)
- [ ] Glassmorphism cards: text on frosted background has sufficient contrast

## Dark Mode Audit
If dark mode is implemented or planned:
- [ ] Glassmorphism cards: `bg-white/10` works in dark mode
- [ ] Text colors: use `text-gray-900 dark:text-gray-100` pairs
- [ ] Border colors: `border-gray-200 dark:border-gray-700`
- [ ] Input backgrounds: `bg-white dark:bg-gray-800`
- [ ] No hardcoded hex colors in Tailwind classes — use semantic tokens
- [ ] System preference detected: `prefers-color-scheme: dark`

## Card Layout Consistency Audit
Every feature page must follow the glassmorphism card pattern:

```typescript
// Standard card pattern
<div className="layer-card p-6 rounded-2xl">
  // Header: title (Sora font) + action buttons
  // Content: table or grid
  // Footer: pagination
</div>
```

- [ ] All feature pages use `.layer-card` class (not raw `bg-white shadow`)
- [ ] Headings: `font-sora` class applied
- [ ] Data numbers: `data-mono` class applied (JetBrains Mono)
- [ ] Status badges: use `.badge` class + variant (success/warning/error/info)
- [ ] Consistent spacing: `p-6` on cards, `gap-4` between cards
