# Play Store Background Location Declaration Checklist

## Critical Architecture Note

> **The official Play Store build MUST be the Capacitor native AAB** generated by
> `.github/workflows/mobile-builds.yml` — NOT a PWABuilder/TWA wrapper.
>
> PWABuilder wraps the app in a Chrome Custom Tab / Trusted Web Activity (TWA).
> When the screen turns off, Android's Doze mode freezes the Chrome renderer process.
> JavaScript timers, `setInterval`, and `navigator.geolocation.watchPosition` all stop.
> There is **no workaround** for this in a browser-based container.
>
> The Capacitor AAB uses `@capacitor-community/background-geolocation` which starts
> a native Android **Foreground Service** with a persistent notification. This is exempt
> from Doze mode and keeps GPS running when the screen is off.

---

## Google Play Background Location Policy

Google requires apps that request `ACCESS_BACKGROUND_LOCATION` to complete a
**Policy Declaration** before their app can be published or updated.

Reference: https://support.google.com/googleplay/android-developer/answer/9799150

---

## Step-by-Step Declaration Process

### 1. Understand the Requirement

- Apps targeting Android 10+ (API 29+) that access location in the background must:
  1. Have a **prominent in-app disclosure** explaining why background location is needed
  2. Submit a **Policy Declaration** in Play Console
  3. Have a **Privacy Policy** covering location data
  4. Optionally provide a **demonstration video**

### 2. In-App Disclosure (already implemented)

The app shows these disclosures:

- **Consent dialog** before first GPS tracking start (implemented in `FieldSalesView.tsx`):
  - "Your GPS location is recorded during your field shift after clock-in"
  - "HR and authorized managers can view your location trail for that day"
  - "Tracking stops when you click End Field Day or clock out"
  - "On the official Android app, tracking continues in background"

- **Foreground service notification** shown while tracking:
  - Title: "Aniston HRMS — Field GPS Active"
  - Body: dynamically shows interval (e.g. "Location recorded every 1h")
  - This is the persistent notification Android requires for foreground services

### 3. Play Console Policy Declaration

Go to: **Play Console → App Content → Sensitive app permissions → Background Location**

Fill in:

| Field | Your Answer |
|-------|-------------|
| **Does your app access location in the background?** | Yes |
| **What is the core feature that requires background location?** | Field Sales Employee Attendance & Route Tracking |
| **Why can't the feature function without background location?** | Field sales employees are required to have their GPS location recorded at configured intervals (e.g., every 60 min) throughout their field shift. If GPS stops when the screen locks, the attendance trail is incomplete and the employee may be marked absent. |
| **How does the app request background location permission?** | Via @capacitor-community/background-geolocation which shows the Android system prompt for "Allow all the time" |
| **Link to your privacy policy** | https://hr.anistonav.com/privacy-policy |

### 4. Privacy Policy Requirements

Your privacy policy at https://hr.anistonav.com/privacy-policy MUST include:

- [ ] What location data is collected (GPS coordinates, timestamp, accuracy)
- [ ] Why it is collected (field employee attendance verification)
- [ ] Who can access it (HR and authorized managers within the organization)
- [ ] How long it is retained (recommended: 90 days active + archive)
- [ ] How employees can request deletion
- [ ] That tracking only occurs during active field shifts (clock-in to clock-out)

### 5. Demonstration Video Checklist

Google may request a video demonstrating the location feature. Record:

- [ ] Open the Aniston HRMS Android app (native APK from Play Store)
- [ ] Log in as a Field Sales employee
- [ ] Navigate to Attendance → Field Sales Tracking
- [ ] Show the consent dialog appearing before first use
- [ ] Tap "I Agree — Start Tracking"
- [ ] Show the foreground service notification in the notification drawer
- [ ] Lock the phone screen
- [ ] Wait 2-3 minutes
- [ ] Unlock the screen
- [ ] Show that GPS points were recorded during the screen-off period
- [ ] Show the GPS trail in HR view (points at the interval)
- [ ] Tap "End Field Day" to stop tracking
- [ ] Show the notification disappears

### 6. AndroidManifest Permissions Verification

The following permissions are patched into `AndroidManifest.xml` during the GitHub Actions build:

```xml
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_BACKGROUND_LOCATION" />
<uses-permission android:name="android.permission.FOREGROUND_SERVICE" />
<uses-permission android:name="android.permission.FOREGROUND_SERVICE_LOCATION" />
<uses-permission android:name="android.permission.POST_NOTIFICATIONS" />
```

Verify after each `npx cap sync` that these are all present in:
`frontend/android/app/src/main/AndroidManifest.xml`

### 7. Pre-Submission Checklist

Before uploading the AAB to Play Console:

- [ ] Verified the AAB is from the Capacitor native build (NOT PWABuilder/TWA)
- [ ] GitHub Actions `build-aab` job completed successfully
- [ ] All 4 signing secrets are configured in GitHub Actions
- [ ] Version code is incremented from the previous upload
- [ ] All AndroidManifest permissions are present
- [ ] Consent dialog tested on real Android device
- [ ] Background GPS tested with screen off for 10+ minutes
- [ ] Privacy policy is live at the URL provided in Play Console
- [ ] Policy Declaration submitted and approved in Play Console
- [ ] Sensitive permissions questionnaire completed

### 8. iOS Background Location (Future)

If an iOS build is planned:

1. Verify `frontend/capacitor.config.ts` has the Geolocation plugin config
2. Add `UIBackgroundModes: ["location"]` to `ios/App/App/Info.plist`
3. Add `NSLocationWhenInUseUsageDescription` and `NSLocationAlwaysUsageDescription` to Info.plist
4. Test that location updates continue after screen lock

**Current status**: iOS build is not fully supported. Do not claim iOS PWA supports
real background GPS — the iOS Safari PWA will pause GPS when the screen locks.
