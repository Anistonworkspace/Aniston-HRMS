# Build Native .aab and .ipa — Aniston HRMS

All publishing files are organized in `store-releases/`.
See `store-releases/PUBLISH_CHECKLIST.md` for the complete guide.

---

## Quick Reference

### Android .aab (Windows — local build)

```
store-releases/android/build-android.ps1
```

Steps:
1. Install Android Studio (bundles JDK 17): https://developer.android.com/studio
2. Copy `signing.keystore` to `store-releases/android/signing.keystore`
3. Copy `store-releases/android/.keystore-env.template` → `.keystore-env`, fill passwords
4. Open PowerShell:
   ```powershell
   cd "C:\Users\aniston user\Desktop\Aniston-hrms\store-releases\android"
   .\build-android.ps1
   ```
5. Upload `store-releases/android/app-release.aab` to Play Console

### iOS .ipa (cloud build via Codemagic)

1. Sign up at https://codemagic.io
2. Connect repo: `Anistonworkspace/Aniston-HRMS`
3. Add Apple Distribution certificate + provisioning profile in Codemagic dashboard
4. Start build → workflow: **Aniston HRMS iOS**
5. Download `app.ipa` from Artifacts

The `codemagic.yaml` in the repo root defines both workflows.

## Version bump before each release

Edit `frontend/android/variables.gradle`:
```groovy
ext {
    versionCode = 2        // must increment on every Play Store upload
    versionName = "1.1.0"  // user-visible version
}
```
